**🍽️ Food Ordering System (MongoDB + Flask)**

**📌 Overview**
This project is a full-stack Food Ordering System developed using MongoDB, Flask, HTML, and CSS. It allows users to browse menus, place orders, and manage their cart across multiple restaurants.
The system was built to understand NoSQL database design, backend integration, and real-world application flow.

**🚀 Key Features**

🧑‍💻 User Flow
* Enter customer details
* Select restaurant (Kuteera / Food Court)
* Choose order type (Dine-in / Takeaway / Delivery)
* Browse menu by categories
* Add multiple items to cart
* Modify cart (update/remove items)
* Place or cancel order
* View final order summary

🍴 Functional Highlights
* Multi-restaurant support
* Dynamic menu handling across categories
* Cart management system
* Delivery charge calculation (5% for delivery orders)
* Order status handling (placed / cancelled)

🛠️ Tech Stack

Frontend
* HTML
* CSS

Backend
* Python (Flask)
* PyMongo

Database
* MongoDB (NoSQL)
* MongoDB Compass

🧠 Database Design

The system uses a structured MongoDB schema with the following collections:
* Restaurants
* Menu_Items
* Customers
* Orders
* Order_Items

Key concepts implemented:
* Document-based data modeling
* Relationships using references
* Aggregation for order summaries
* CRUD operations for order lifecycle


## 📈 Key Learnings

* Integrating MongoDB with Flask using PyMongo
* Building a complete end-to-end application flow
* Handling real-world scenarios like cart updates and order management
* Understanding differences between SQL and NoSQL systems
